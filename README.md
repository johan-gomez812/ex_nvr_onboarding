30-day guide to learning Elixir, Nerves, and Video at Evercam.

# 🎥 ex_nvr Junior Developer Roadmap

Welcome! This repository is your guide to becoming an expert developer in the **Evercam ex_nvr** stack. In 30 days, you will go from knowing nothing about Elixir to building your own monitoring system.

## 🤖 "AI-First" Approach
At Evercam, we don't just write code; we use intelligence to move faster.
1. Install **VS Code** with the `ElixirLS` extension.
2. Configure [Claude Code](https://github.com/anthropics/claude-code) in your terminal.
3. **The Rule:** Before asking a human, use Claude to explain concurrency concepts or `ex_nvr` internals.

---

## 📅 The Action Plan

### Phase 1: The 7-Day "Immersion"
* **Day 1-2:** Elixir Koans and functional syntax. (Use Claude to refactor `for` loops into `Enum.map`).
* **Day 3-4:** Nerves & RPi. Learn how to burn firmware and understand why Nerves is better than standard Linux.
* **Day 5-7:** `ex_nvr` Deployment. Clone the main repo and get it working with a virtual camera (FFmpeg).

### Phase 2: The 30-Day "Specialization"
* **Week 2:** [Membrane Framework](https://membrane.stream/learn). Understand how video flows through bins.
* **Week 3:** Phoenix LiveView. Create a dashboard that shows camera status in real-time.
* **Week 4:** Final Project. Implement simple "Motion Detection" logic using an IP camera.
